# Publish Commands for @mariozechner/pi-tui-professional v1.0.0

## Prerequisites
- npm account with ownership of @mariozechner scope
- 2FA enabled (recommended)
- Logged in to npm

## Quick Publish

```bash
cd packages/tui
npm publish --access public
```

That's it. The prepack hook will run build + test + dry-run automatically.

---

## Detailed Step-by-Step

### 1. Check npm login
```bash
npm whoami
```
Should output your npm username. If error, run `npm login`.

### 2. Final validation (optional, prepack does this)
```bash
npm run build
npm test
npm pack --dry-run
```

### 3. Publish
```bash
npm publish --access public
```
- Enter 2FA code when prompted (if enabled)
- Wait for upload to complete

### 4. Verify
Visit: https://npmjs.com/package/@mariozechner/pi-tui-professional
- Version should show 1.0.0
- Files tab should show dist/, README.md, LICENSE, etc.

### 5. Git tag
```bash
git tag -a v1.0.0 -m "Release v1.0.0"
git push origin v1.0.0
```

### 6. Create GitHub release
- Go to: https://github.com/qcoder/qclaw/releases
- Click "Draft a new release"
- Tag: v1.0.0
- Title: v1.0.0
- Description: Copy from CHANGELOG.md
- Publish

### 7. Test installation
```bash
mkdir ~/test-publish && cd ~/test-publish
npm init -y
npm install @mariozechner/pi-tui-professional
npx tsx -e "import { ThemeManager } from '@mariozechner/pi-tui-professional'; console.log('OK')"
```

## Troubleshooting

### "You do not have permission to publish"
- Verify you own the @mariozechner scope: `npm access ls-packages @mariozechner`
- If part of a team, ask admin to grant publish rights: `npm access grant read-write @mariozechner <username>`

### "prepack script failed"
Fix the failing command locally. Common issues:
- Build errors: run `npm run build` and fix TypeScript errors
- Test failures: run `npm test` and address failing tests

### "Package name already exists"
The package already exists on npm. Options:
- Bump version and publish new: `npm version patch && npm publish`
- If you need to replace within 72h: `npm unpublish @mariozechner/pi-tui-professional@1.0.0` then republish

### " ENEEDAUTH"
Not logged in. Run `npm login` first.

## Rollback (if needed within 72h)

```bash
npm unpublish @mariozechner/pi-tui-professional@1.0.0
```

After 72h, use deprecation:
```bash
npm deprecate @mariozechner/pi-tui-professional@1.0.0 "Critical bug. Upgrade to v1.0.1."
```

Then publish fix: `npm version patch && npm publish`
